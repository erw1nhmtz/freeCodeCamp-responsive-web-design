Introduction

Web accessibility refers to the inclusive practice of ensuring there are no barriers that prevent interaction with, or access to, websites on the World Wide Web by people with disabilities.

When sites are correctly designed, developed and edited, generally all users have equal access to information and functionality.

This documentation covers key aspects of web accessibility that developers should consider when building websites.

// Example of an accessible button <button type="button" aria-label="Close" onclick="closeModal()">X</button>

Semantic HTML

Semantic HTML is HTML that introduces meaning to the web page rather than just presentation. Examples of semantic HTML tags include:

  • <header>
  • <nav>
  • <main>
  • <article>
  • <aside>

Using semantic HTML helps screen readers and other assistive technologies understand the structure of your web page.

Here's an example of semantic HTML structure:

<header> <h1>Website Title</h1> <nav> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> <li><a href="#">Contact</a></li> </ul> </nav> </header>

ARIA

ARIA (Accessible Rich Internet Applications) is a set of attributes that define ways to make web content and web applications more accessible to people with disabilities.

ARIA supplements HTML so that interactions and widgets commonly used in applications can be passed to assistive technologies.

Here are some common ARIA attributes:

  • aria-label
  • aria-labelledby
  • aria-hidden
  • role

Example of ARIA in use:

<button aria-label="Close" onclick="closeModal()">X</button>

Keyboard Navigation

Keyboard navigation is crucial for users who can't use a mouse. All interactive elements should be keyboard accessible.

Ensure that users can tab through your site in a logical order.

Use the tabindex attribute to control the order of focus:

<div tabindex="0">This div is keyboard focusable</div>

Remember to style focus states to make it clear which element is currently focused:

:focus { outline: 2px solid blue; }

Color Contrast

Sufficient color contrast between text and its background is important for users with visual impairments.

The Web Content Accessibility Guidelines (WCAG) 2.1 recommend the following contrast ratios:

  • At least 4.5:1 for normal text
  • At least 3:1 for large text

You can use online tools to check your color contrast ratios.

Here's an example of CSS with good color contrast:

body { color: #333; /* Dark gray text */ background-color: #fff; /* White background */ }
