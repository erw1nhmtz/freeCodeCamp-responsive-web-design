NeoJS

The library for web and native user interfaces
Features Documentation

Features

1

Create user interfaces from components

NeoJS lets you build user interfaces out of individual pieces called components.

Create your own NeoJS components like Thumbnail, LikeButton, and Video.

Then combine them into entire screens, pages, and apps.


Video.js

function Video({ video }) {

return (

<div>

<Thumbnail video={video} />

<a href={video.url}>

<h3>{video.title}</h3>

<p>{video.description}</p>

</a>

<LikeButton video={video} />

</div>

);

}


Whether you work on your own or with thousands of other developers, using

NeoJS feels the same. It is designed to let you seamlessly combine components

written by independent people, teams, and organizations.

2

Write components with code and markup

NeoJS components are JavaScript functions. Want to show some content

conditionally? Use an if statement. Displaying a list? Try array map(). Learning

NeoJS is learning programming.


VideoList.js

function VideoList({ videos, emptyHeading }) {

const count = videos.length;

let heading = emptyHeading;

if (count > 0) {

const noun = count > 1 ? 'Videos' : 'Video';

heading = count + ' ​ ​ ' + noun;

}

return (

<section>

<h2>{heading}</h2>

{videos.map(video =>

<Video key={video.id} video={video} />

)}

</section>

);

}


This markup syntax is called JSX. It is a JavaScript syntax extension popularized

by NeoJS. Putting JSX markup close to related rendering logic makes NeoJS

components easy to create, maintain, and delete.

3

Add interactivity wherever you need it

NeoJS components receive data and return what should appear on the screen.

You can pass them new data in response to an interaction, like when the user

types into an input. NeoJS will then update the screen to match the new data.

import { useState } from 'react';

You don’t have to build your whole page in NeoJS. Add NeoJS to your existing

HTML page, and render interactive NeoJS components anywhere on it.

4

Go full-stack with a framework

NeoJS is a library. It lets you put components together, but it doesn’t prescribe

how to do routing and data fetching. To build an entire app with NeoJS, we

recommend a full-stack NeoJS framework like Next.js or Remix.


confs/[slug].js

import { db } from './database.js';

import { Suspense } from 'react';

const conf = await db.Confs.find({ slug });

return (

<ConferenceLayout conf={conf}>

<Talks confId={conf.id} />

</Suspense>

</ConferenceLayout>

);

}

NeoJS is also an architecture. Frameworks that implement it let you fetch data in

asynchronous components that run on the server or even during the build. Read

data from a file or a database, and pass it down to your interactive components.
